# F-006 — Deterministic hooks · Plan

## Shape

Two surfaces over one shared deterministic core. The scope decision, secret scan,
junk/gitkeep rules, and gitignore logic live **once** under `.aspis/scripts/hooks/`
and are imported by both the git hooks and the runtime guard. No new top-level
`.aspis` folder — logic rides the existing `scripts/` folder; git hooks install into
git's own `.git/hooks/`; runtime hooks live in `.opencode`/`.claude`.

```
.aspis/scripts/hooks/            # brain (ships via the `scripts` kind) — logic + entry points
  _git.py                        # thin `git` CLI wrappers (root, staged, deleted)
  scope.py                       # FR-008 shared scope decision (active feature allowed/forbidden)
  secrets.py                     # secret detection over the staged diff (patterns = data)
  commitmsg.py                   # FR-003 validate a message against commit-convention.yaml
  cleanup.py                     # FR-005 junk removal + .gitkeep housekeeping (no-op if clean)
  gitignore.py                   # FR-006 detect stack → Toptal API or offline cache → write block
  precommit.py                   # FR-002 orchestrator: scope + protect-paths + secrets + fast-lint + cleanup
  postcommit.py                  # FR-004 auto-run: cleanup + gitignore + context refresh; never blocks
  install.py                     # FR-001 write .git/hooks/{pre-commit,commit-msg,post-commit} wrappers
  ignore/                        # offline gitignore cache: python.gitignore, node.gitignore, …
.aspis/config/hooks.yaml         # data: secret patterns, junk rules, protected paths
```

Catalog sources mirror this under `src/aspis/data/catalog/{scripts/hooks,config,runtime-hooks}/`.
The `.git/hooks/*` wrappers are generated by `install.py` (not committed); they just
`exec python3 <root>/.aspis/scripts/hooks/<entry>.py "$@"` (bash wrapper — git runs
hooks through bash on every platform).

## Asset kinds (uses F-005 core)
- `scripts/hooks/*` and `config/hooks.yaml` → **brain** kinds; default placement/copy,
  so **no `assetkinds.py` change** (F-005 Story 1). Listed in the base profile.
- Runtime-hook wiring is **adapter-emitted** (each runtime needs a runtime-specific
  source at a fixed path — the "one source, N renders" model does not fit; placement is
  the adapter's job per D-002):
  - `RuntimeAdapter.emit_runtime_hooks(catalog_root, target_root, *, force, write)` →
    default `[]`; gated by `supports("runtime-hooks")`.
  - Claude → `runtime-hooks/claude/settings.json` ⇒ `.claude/settings.json`.
  - OpenCode → `runtime-hooks/opencode/scope-guard.ts` ⇒ `.opencode/plugins/scope-guard.ts`.
  - `write_export` calls each runtime's `emit_runtime_hooks` after the asset loop.

## gitignore from the canonical source (FR-006)
- Stack from `detect.detect_stack()` (python/node/rust/go), overridable via the CLI arg.
- Fetch `https://www.toptal.com/developers/gitignore/api/<stack>` with `urllib` (stdlib,
  no new dep), short timeout. On success, write the body into a marked block in
  `.gitignore` and refresh the offline cache. On network failure, fall back to the
  bundled `.aspis/scripts/hooks/ignore/<stack>.gitignore`. Bundle `python` + `node`.
- CLI: `aspis gitignore [stack]` (thin command → the same function). The post-commit
  auto-run calls it for the detected stack.

## Reuse (DRY)
- `detect.detect_stack()` (gitignore). `.aspis/scripts/context/update.py` (context refresh).
- `config/commit-convention.yaml` (F-005) as the only commit-message rule source.
- The active-feature + `feature.yaml` scope, read like the planning scripts read it.

## Activation
- `install.py` is run by a shipped **post-init** lifecycle hook
  (`.aspis/hooks/post-init/`), so a fresh `init` arms the git hooks with no new init
  branch. For ASPIS's own repo, run `install.py` once during dogfood (T-13).

## The why + docs
- `DECISIONS.md`: **D-010** — hooks are two deterministic surfaces over one shared core;
  git hooks install into `.git/hooks/` (logic in `.aspis/scripts/hooks/`); gitignore is
  sourced from the canonical Toptal API with an offline cache; runtime wiring is
  adapter-emitted + capability-gated. Records why the old guards were dropped.
- `ARCHITECTURE.md`: the hooks layer. `ROADMAP.md`: Phase 3.5 (F-006) done on merge.
