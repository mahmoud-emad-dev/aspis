# F-012 — Implementation Plan

## Summary
Make the commit-message convention self-healing and auditable without blocking
the developer. Three slices: (1) a **non-blocking auto-fix** in the commit-msg
hook that strips unambiguous forbidden attribution and lets the commit proceed;
(2) a **history audit** shared by `aspis doctor` (read-only summary) and a new
`aspis commits` verb; (3) **config-driven** auto-fix policy + a per-commit escape
hatch. All rules stay in `commit-convention.yaml` / `hooks.yaml` (R-007); the
hook gains behaviour, not rules. Story 4 (learnable style) is deferred to F-013.

## Technical context
- **Language / version**: Python 3.11+ (matches the repo; runs on 3.12/3.14 too).
- **Key dependencies**: stdlib + PyYAML + the existing `_git`/`_config` hook
  helpers. No new third-party deps.
- **Storage / interfaces**: git commit-message file (commit-msg hook arg),
  `commit-convention.yaml`, `hooks.yaml`; git history via `_git` / subprocess.
- **Testing**: pytest; red→green per R-005. New `tests/test_commits.py`, extend
  `tests/test_commitmsg.py` and the doctor test.
- **Project type / structure**: hook logic in
  `src/aspis/data/catalog/scripts/hooks/commitmsg.py` (catalog SSoT; the
  `.aspis/scripts/hooks/` copy is regenerated by export). New CLI verb in
  `src/aspis/commands/commits.py`, registered via `commands/__init__.py`
  `COMMAND_MODULES` (cli.py never changes). Doctor check in
  `src/aspis/commands/doctor.py`.
- **Constraints**: C-PORTABLE (UTF-8 message I/O, pathlib, no shell assumptions;
  Windows = gate of record); deterministic pure transforms; F-007 single-writer
  git for any history rewrite.

## Components
1. **Shared attribution detector** — refactor the token + credited-model matching
   currently inside `commitmsg.validate()` into one helper
   `find_attribution(message, convention) -> list[(lineno, line)]`, so `validate`
   (report) and the new `autofix` (repair) share a single source of truth.
2. **Auto-fix transform** — `strip_attribution(message, convention) -> (cleaned,
   removed)`: pure; drops only the flagged lines, collapses orphaned blank lines,
   byte-stable when nothing matches. `commitmsg.main()` calls it first, rewrites
   the message file when `removed`, prints `auto-fix: removed …`, then validates
   the cleaned message (remaining advisory issues stay warn). Honours the
   `Commit-Style: skip` escape hatch and the `autofix.*` policy.
3. **Config** — add an `autofix` block to `commit-convention.yaml`
   (`attribution: true` default; room for future repairs) and document the
   `Commit-Style: skip` trailer. Read through `_config`.
4. **History audit (read)** — `audit_history(root, convention) -> list[Report]`
   in a small shared module (e.g. `src/aspis/commitaudit.py`), iterating
   `git log` messages and running `validate`. Consumed by:
   - `aspis doctor` → a `[ok]/[warn] commits` line with the count.
   - `aspis commits --audit` → the per-commit detail + totals (exit 1 on findings).
5. **History fix (write)** — `aspis commits --fix`: human-invoked; create a backup
   ref `backup/commits-fix-<date>`, rewrite only flagged messages (reword), assert
   tree/content byte-identical, never touch already-pushed history without the
   explicit flag. Mirrors the manual recovery used on 2026-06-21.
6. **Export parity** — after editing the catalog hook, regenerate the shipped
   `.aspis/scripts/hooks/commitmsg.py` (export copy) so live == catalog.

## Risks & rollback
- **Risk**: auto-fix strips a legitimate line → only unambiguous attribution
  patterns are removed (the F-008 context-aware matcher); false-positive tests
  guard `.claude`/`.opencode` domain mentions.
- **Risk**: history `--fix` corrupts content → backup ref first, assert
  content byte-identical, human-invoked only, never auto-run.
- **Risk**: catalog/live hook drift → export regenerates the copy; step 7 asserts
  parity (the manual-regen debt is known — keep it in one step).
- **Rollback**: revert the feature branch; the hook returns to warn-only validate.
  No data migration, no stored state.
