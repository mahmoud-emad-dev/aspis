## Findings

| ID | Severity | Location | Description | Recommendation |

| F-002 | NIT | `PLAN.md:522-557` (§9 Function Signatures) | The "complete function signatures" block for `src/aspis/protect.py` lists `sha256_text`, `decide`, `plan_runtime`, and `summary`, but omits `sha256_bytes` even though `sha256_bytes` is part of the public interface declared in PLAN §2.1 and implemented at `protect.py:74-80`. | Add `def sha256_bytes(data: bytes) -> str: ...` to the §9 signature list so the plan remains internally consistent. |

## Test Quality Assessment

- **Coverage:** The 45 tests cover all six decision cases, ordering edge cases (first-match-wins), `sha256_text` BOM/CRLF/bare-CR normalization, `sha256_bytes` raw-byte behaviour, `plan_runtime` key-union fan-out, and `summary` aggregation.
- **Correctness of constants:** Hardcoded SHA-256 values for empty string (`e3b0c4...`), `"abc"` (`ba7816b...`), and `"hello world"` (`b94d27b...`) were cross-checked and are correct.
- **Behaviour-only:** Tests call public functions and assert on returned values; they do not inspect private state or monkey-patch internals. The frozen-dataclass test uses a generic `Exception` catch, which is acceptable.
- **Pass rate:** 45 passed, 0 failed, 0 errors, 0 skipped (independently verified).
- **Minor gap:** `plan_runtime` is not explicitly exercised with a `live_hashes` dict that contains an explicit `None` value (as opposed to an absent key). This is not a correctness gap because `.get(path)` returns `None` in both cases, but a future regression test could add it for completeness.
